Backup

Gera um arquivo único com todos os dados da plataforma (cardápios, pedidos, sessões e contas).

⚠️ No Fly.io, a pasta de backups é efêmera (fora do volume). Se o container reiniciar, o arquivo some. Gere e baixe o backup na mesma sessão.

Como restaurar

A restauração é manual, feita com o servidor parado. Este painel não executa a restauração — siga o passo a passo abaixo.

A restauração substitui os dados atuais. Pare o app antes e mantenha um resguardo (data.old) até confirmar que deu certo.

No Fly.io

# 1) Suba o arquivo do seu PC para o container
fly ssh sftp shell
  put backup-AAAA-MM-DD-HHmm.tar.gz /app/backups/
  exit

# 2) Pare o processo e restaure
fly ssh console
  cd /app
  mv data data.old          # resguardo (não apague ainda)
  mkdir data
  tar -xzf backups/backup-AAAA-MM-DD-HHmm.tar.gz -C data
  ls data data/tenants      # confira a estrutura
  exit

# 3) Reinicie a máquina
fly machine restart <machine_id>

# 4) Valide login + bot. Se tudo ok:
fly ssh console -C "rm -rf /app/data.old"

Localmente (PM2 / VPS)

pm2 stop bot-restaurante      # ou Ctrl+C no npm start
mv data data.old
mkdir data
tar -xzf backup-AAAA-MM-DD-HHmm.tar.gz -C data
pm2 start bot-restaurante     # ou npm start
# valide e depois: rm -rf data.old

O .tar.gz guarda o conteúdo de data/ na raiz, então tar -xzf … -C data recoloca tudo no lugar. As sessões baileys-*/ voltam juntas — o bot reconecta sem novo QR (a menos que o WhatsApp já tenha expirado a sessão).

Excluir restaurante

Esta ação é irreversível. Apaga o cardápio, todos os pedidos e a sessão de WhatsApp deste restaurante. Não há como desfazer.
